loci-extract

Upload documents to extract text, tables, and structured data. Queue multiple files for batch processing. Files are processed in memory and deleted immediately.

Drag and drop one or more files, or click to browse. Supported: PDF, DOCX, XLSX, PNG, JPG, TIFF, WEBP, TXT, MD, CSV, JSON. Max 25 MB each.

2. Choose extraction pipeline

Pick how documents should be processed. The VLM pipeline sends page images directly to an AI vision model — no OCR needed. The OCR pipeline uses traditional text extraction with optional GPU acceleration.

How Identify works: Renders page 1 as an image and asks the VLM to identify the document type. Returns the document type, tax year, entity name, and a suggested filename for sorting/renaming. No text extraction is performed — this is designed to be fast for batch document sorting. Falls back to rule-based classification if VLM is unavailable.

No additional settings needed. Documents will be identified and results will show type, year, entity, and suggested filename.

Debug mode: Drop one file and this runs it through every selected pipeline. The result modal gives you a tab per pipeline with the raw text, a VLM stage trace, and a Diff Text toggle so you can tell exactly which stage is degrading output. Pipelines that need a backend your host doesn't have (e.g. PaddleOCR without a GPU) will be marked unavailable instead of failing the whole run.

Pipelines run sequentially on the server (VLM is the slow path — the 32B model can take 30–90s per page). Only the first file in the queue is processed when Compare is selected.

How VLM Hybrid works: For each PDF page: (1) reads the text layer, (2) uses a fast LLM to verify the text is real (not garbage), (3) if text is bad or missing, renders the page as an image and sends it to the VLM. Clean pages are instant; only bad pages get the full VLM treatment. This gives you speed on good PDFs and accuracy on garbage ones.
Hybrid mode: reads text layer first, verifies quality with fast LLM, only renders page images for VLM when text is bad. Fast on clean PDFs, accurate on garbage ones.
Structured mode classifies the document and extracts typed fields (W-2 boxes, receipt totals, etc). Raw mode returns text only.
Optional. If you know the document type, this helps the model focus on the right fields.
Profiles bundle OCR, classification, and enrichment settings for a document family.
Raw = canonical text + segments. Structured = also classifies doc type and extracts typed fields.
Auto = text layer first, OCR if empty. Force Image = ignore junk text layers, OCR the rendered image.
Auto prefers PaddleOCR on GPU. Tesseract works on CPU.
Chunks are sized text blocks for retrieval pipelines.
Override auto-detection if you know the type. Structured mode only.

3. Processing queue

How it works

Batch Queue

Drop multiple files at once. They process sequentially through the server. Each file shows real-time status: pending, processing, done, or error. Click any completed file to inspect results.

Profiles

Tax uses Donut IRS classifier (28 form types, GPU). Financial enables layout analysis + LLM section labeling. Receipt forces OCR for scans. General is rules-only, no GPU needed.

Raw vs Structured

Raw: raw_text, segments, chunks. Structured: adds classification (doc type) and structured.fields (typed data like W-2 boxes, receipt totals).

OCR & GPU

PaddleOCR runs on GPU, fast + accurate. Tesseract is CPU with multi-pass. Auto strategy OCRs only when PDF has no text layer.

Ephemeral

Files upload, process in memory, and delete. Results exist only in your browser. Download JSON or text before closing the page.

API Access

This UI calls the same API. See /docs for Swagger, or curl -X POST /extract -F "file=@doc.pdf" from CLI.
